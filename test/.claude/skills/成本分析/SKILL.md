# 成本分析技能

根据新华医院月度运营分析 Excel 底稿，自动更新 PPT 报告中成本分析模块的表格数据和文本叙述段落。

## 输入

- **Excel 底稿**：如 `4月分析底稿.xlsx`
- **PPT 模板**：如 `新华医院2026年4月运营分析0509.pptx`
- **卫生材料明细**（自动检测）：同目录下 `*卫生材料明细.xlsx`（如 `2026年1-4月卫生材料明细.xlsx`）

## 执行

```bash
python .claude/skills/成本分析/scripts/cost_analysis.py <excel_path> <pptx_path> [--output OUTPUT]
```

示例：
```bash
python .claude/skills/成本分析/scripts/cost_analysis.py 4月分析底稿.xlsx 新华医院2026年4月运营分析0509.pptx
```

无 `--output` 参数时，输出文件自动命名为 `<原名>_updated_<MMDD>.pptx`。

脚本自动从 Excel 文件名提取数据月份（如 `4月分析底稿.xlsx` → 4），用于时间节点计算和明细数据期间过滤。

## 数据规则

| Excel Sheet 类型 | 用途 | 是否 ÷ 10000 |
|---|---|---|
| 前端分析 sheet（医疗成本、人员经费等，已是万元） | 成本分类汇总 | 否 |
| 附表 sheet（本月附表1/2/3 等） | 成本明细、收支余 | 是 |
| 数据表 sheet（当年/去年累计数据表） | 收入数据 | 是 |
| 当年连续累计数据表 | 累计汇总数据 | 是 |
| 同比/环比 sheet | 同期比较数据 | 是 |
| 卫生材料明细文件（独立 Excel） | 卫生材料逐行明细聚合 | 是 |

## 更新范围

### 表格数据更新（Slide 4, 50-75）

仅更新成本分析模块中关键汇总指标的表格数据，包括：

- **Slide 4**：收支结余情况表（医疗收入从当年/奉贤/去年累计数据表取数，医疗成本从附表2取数，财政基本拨款收入不变，医疗服务盈余 = 医疗收入+财政拨款-医疗成本，其他盈余 = 收支结余-医疗服务盈余，收支结余从附表1医疗业务盈余取数）
- Slide 50：医疗成本结构表（人员经费、药品费、卫生材料费、折旧/摊销、医疗风险基金、商品和服务费、医疗成本合计）
- Slide 51：人员经费明细
- Slide 52：卫生材料费明细（可收费、不可收费、检验试剂、其他不可收费从明细文件聚合取数）
- Slide 53：检验试剂成本与化验收入（检验试剂成本、医学检验科从明细文件聚合取数；含门急诊人次、住院人次）
- Slide 54：商品和服务费用
- Slide 55：能源成本（含杨浦院区计算列）
- Slide 56：物业管理费
- Slide 57：维修护费
- Slide 59：卫生材料综合分析
- Slide 65：预算完成情况（跨 sheet 取数，工资总额自动汇总）
- Slide 75：成本指标目标完成情况

### 文本段落数据替换（Slide 4, 51-57, 65）

同时更新各页文字叙述段落中引用的数字，通过模板变量 `{var_name}` 替换实现。文本 shape 按名称匹配（如 `矩形 8`、`矩形 7`、`矩形 13`）。

**不更新**：图表、科室明细（Slide 60-63）。

## 新增功能

### 1. 文本段落替换

通过 `texts` 配置在 mapping.json 中定义每页的文本模板、变量绑定和格式覆盖。脚本更新表格后，自动查找对应名称的 shape 并替换其中的数字。

支持的格式类型：`int`（整数）、`wan_int`（万元整数）、`wan_1dp`（万元一位小数）、`pct_1dp`（百分比一位小数）、`pct_2dp`（百分比两位小数）、`pp_1dp`（百分点一位小数）、`pct_abs_1dp`（百分比绝对值一位小数）。

### 2. 计算列与计算变量

通过 `ComputedValueResolver` 支持以下计算类型：

| 类型 | 说明 | 示例 |
|---|---|---|
| `subtract` | 两列相减 | 杨浦院区 = 合计 - 奉贤 |
| `add` | 两列相加 | |
| `add_refs` | 两独立数据源相加 | 医疗收入 + 财政拨款 |
| `divide` | 两列相除 | 执行率 = 执行数 / 核定数 |
| `multiply` | 两列相乘 | |
| `sum` | 跨行跨 sheet 求和 | 工资总额 = 基本工资 + 津贴补贴 + 伙食补助费 + 绩效工资 + 其他 |
| `time_node_diff` | 执行率 - 月份/12 | 执行率与时间节点差异 |
| `pct_diff` | 同比变化率 | (当期 - 同期) / 同期 |
| `pct_change` | 增减率 | 增减额 / (当期 - 增减额) |
| `divide_excel` | 从 Excel 直接取两值相除 | 文本变量中的执行率计算 |
| `time_node_diff_excel` | 从 Excel 取值计算时间节点差异 | 文本变量中的时间节点差异 |
| `subtract_excel` | 从 Excel 取值相减 | 文本变量中的差值 |
| `sum_excel` | 从 Excel 跨行求和后除以 10000 | 文本变量中的工资总额 |
| `detail_sum` | 从明细文件按条件聚合求和 | 不可收费 = SUM(是否收费=否) |
| `subtract_refs` | 独立解析两个数据源后相减 | 可收费 = 卫生材料费 - 不可收费 |

### 3. 表格列格式覆盖

`cols` 字典格式条目可指定 `"format"` 字段，强制覆盖自动格式推断：

- `"2dp"`：保留两位小数（用于人次、比例等）
- `"int"`：四舍五入到整数

```json
{ "ppt": 1, "source": "当年数据表", "xl_label": "小计", "xl": 6, "format": "2dp" }
```

### 4. 混合数据源

同一行不同列可从不同 Excel sheet 取数。`cols` 支持两种格式：

**双区域 sheet 处理**：部分 sheet（如 附表2）采用左右双区域布局，左侧为收入明细、右侧为成本明细。通过创建多个 excel_sources 别名指向同一 sheet 但使用不同的 `label_col` 来分别索引两个区域（如 `当月附表2` label_col=0 索引收入区域，`当月附表2-成本` label_col=3 索引成本区域）。

- **旧格式**（列表）：`[ppt_col, xl_col]` — 使用行默认 source
- **新格式**（字典）：`{"ppt": N, "xl": M, "source": "...", "compute": {...}, "format": "..."}`

处理分两遍：第一遍解析直接取值列，第二遍解析计算列（可引用第一遍的结果）。

### 5. 明细数据聚合取数

脚本自动检测同目录下的 `*卫生材料明细.xlsx` 文件，通过 `MaterialDetailLoader` 加载。

明细文件结构：
- **当期 sheet**（20列）：含院区列（杨浦院区/奉贤院区）
- **去年同期 sheet**（19列）：无院区列

`detail_sum` 支持的过滤条件：

| 参数 | 说明 |
|------|------|
| `year` | 数据年份（2026/2025） |
| `chargeable` | 是否收费（`true`=是, `false`=否） |
| `subject_contains` | 入账科目包含文本（如 `"化验材料费"`） |
| `dept_contains` | 部门名称包含文本（如 `"医学检验科"`） |
| `campus` | 院区过滤（如 `"奉贤院区"`，仅2026数据有效） |
| `needs_wan` | 是否 ÷10000 |

期间过滤自动根据 `data_month` 和 `year` 生成（如 4月→`"2026-04"`、`"2025-04"`），无需手动配置。

### 6. 自动月份检测

`detect_data_month()` 从 Excel 文件名提取月份数字，用于时间节点计算（当月/12）和明细数据期间过滤。

## 映射配置

数据映射关系在 `references/mapping.json` 中定义。每个 slide 的映射包含：

- **excel_sources**：Excel sheet 别名 → sheet 名称、是否 ÷10000、标签列
- **slides**：每页配置
  - `headers` + `header_row`：表格定位方式（通过表头文本匹配）
  - `rows`：每行配置
    - `label`：行标签（模糊匹配 PPT 表格第 0 列）
    - `source`（可选）：默认数据来源（excel_sources 的 key）
    - `xl_label`（可选）：Excel 中对应的行标签（默认与 label 相同）
    - `cols`：列映射（支持列表格式和字典格式，含 `format` 和 `compute` 字段）
  - `texts`（可选）：文本段落配置
    - `shape_name`：PPT shape 名称
    - `template`：Python format 模板字符串
    - `vars`：变量 → 数据源绑定
    - `format_overrides`：变量格式覆盖

## 关键业务逻辑

### 收支结余（Slide 4）
- **医疗收入**：从 `当年数据表`（当期）、`当年累计奉贤`（奉贤院区）、`去年数据表`（上年同期）取数（均为小计行、医疗收入列 index 14）
- **财政基本拨款收入**：不更新，保持 PPT 原有数据不变
- **医疗成本**：从 `当月附表2-成本`（当期）、`当月奉贤附表2-成本`（奉贤）、`上年同期附表2-成本`（上年同期）取 `一、医疗成本` 行当月值（label_col=3 定位成本区域）
- **医疗服务盈余**：通过 `subtract_refs` + `add_refs` 嵌套计算 = 医疗收入 + 财政拨款 - 医疗成本
- **收支结余**：从 `当月附表1`（当期）、`当月附表1-奉贤`（奉贤）、`上年同期附表1`（上年同期）取 `一、医疗业务盈余` 行当月值
- **其他盈余**：通过嵌套 `subtract_refs` 计算 = 收支结余 - 医疗服务盈余
- **文本段落（矩形 8）**：更新当月和1-4月累计的收支结余数据（医院/杨浦/奉贤），杨浦通过 `subtract_excel` 计算（医院 - 奉贤）

> 附表2 为双区域布局（左收入右成本），`附表2-成本` 源使用 label_col=3 索引成本区域。
> 财政基本拨款收入数值保留 PPT 原始数据，不作为更新目标。

### 预算完成情况（Slide 65）
- **医疗收入 执行数**：从 `当年连续累计数据表` 取数（O 列 index 14），非预算完成情况 sheet
- **医疗成本 执行数**：从 `当月附表3` 累计数列取数
- **工资总额 执行数**：基本工资 + 津贴补贴 + 伙食补助费 + 绩效工资 + 其他工资福利支出（从当月附表3 累计数列取各组件 raw 值，求和后 ÷10000）
- **执行率**：执行数 / 核定数（核定数仍从预算完成情况 sheet 取）
- **时间节点**：数据月份 / 12（如 4月 → 33.33%），较时间节点 = 执行率 - 时间节点

### 能源成本（Slide 55）
- **杨浦院区**（PPT col 3）：当期合计 - 奉贤院区（通过 `subtract` 计算列实现）

### 卫生材料费明细（Slide 52）

以下行从 `卫生材料明细.xlsx` 聚合取数（仅取对应月份数据）：

| 行 | 取数规则 |
|----|---------|
| **不可收费** | `detail_sum`: SUM(是否收费=否) |
| **可收费** | `subtract_refs`: 卫生材料费 - 不可收费 |
| **检验试剂** | `detail_sum`: 是否收费=否 + 入账科目含"化验材料费" |
| **其他不可收费卫生材料** | `subtract_refs`: 不可收费 - 检验试剂 |

### 检验试剂成本（Slide 53）

| 行 | 取数规则 |
|----|---------|
| **检验试剂成本** | 同 Slide 52 检验试剂 |
| **医学检验科** | `detail_sum`: 是否收费=否 + 入账科目含"化验材料费" + 部门名称含"医学检验科" |
| **门急诊人次** | 从 `当年数据表` G 列（index 6）取数，保留两位小数 |
| **住院人次** | 从 `当年数据表` H 列（index 7）取数，保留两位小数 |

> 奉贤院区数据仅当期（2026）可用，去年同期（2025）明细文件无院区列，奉贤去年同期不更新。
> 人数列通过 `"format": "2dp"` 配置保留两位小数。

## 验证

运行后检查输出摘要：
- `updated_cells`：成功更新的单元格数
- `skipped_rows`：未匹配到的行数（预期为 0，若非 0 需检查 Excel/PPT 标签是否变化）
- `skipped_cells`：未解析到的单元格数
- 文本 shape 更新日志：`[UPD] Text shape '矩形 X' updated`
- 明细文件加载：`[DETAIL] Found: ...` 和 `[LOAD] 卫生材料明细 2026/2025 ...`
- 交叉验证：可收费 = 卫生材料费 - 不可收费，不可收费卫生材料 = 不可收费 - 检验试剂
- 抽查 Slide 65 的医疗收入执行数、执行率是否与 Excel 数据一致

## 扩展

如需增加新的数据映射：
1. 在 `references/mapping.json` 的 `excel_sources` 中添加新的 Excel sheet 别名
2. 在 `slides` 中添加新的 slide 映射配置（表格行 + 文本段落）
3. 如需新的计算类型，在 `cost_analysis.py` 的 `ComputedValueResolver.resolve()` 中添加
4. 如需新的明细聚合过滤条件，在 `MaterialDetailLoader.aggregate()` 中添加
5. 重新运行脚本测试
